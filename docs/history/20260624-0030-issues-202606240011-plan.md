## 目的（課題）

1. 半径円が地図・PNG に表示されない（ズーム過大で円が画面外）→ 半径に応じ動的調整。
2. 企業一括データを店舗ごとCSVのZIPではなく、対象企業＋半径で絞った単一CSV（cp932）に。
3. 企業一括を「データ抽出」と「画像抽出」の2機能に分割。
4. `pages/` による Streamlit 自動マルチページ（app/help/map_bulk/map_single 表示）を除去し、
   ラジオのみで切替。

## 対象ファイル

- `lib/data.py`: `zoom_for_radius` を動的算出に変更、`filter_company` 追加。
- `lib/map_builder.py` / `lib/static_map.py`: 新ズーム呼出（viewport=560 / size）。
- `lib/zip_builder.py`: `build_csv_zip` 削除（`build_png_zip` 維持）。
- `pages/` → `views/`（改名）。`views/bulk_data.py`（SC-02a）・`views/bulk_image.py`（SC-02b）
  を新設し、旧 `pages/map_bulk.py` を廃止。`map_single.py`/`help.py` は移設のみ。
- `app.py`: radio 4項目化＋`views` ディスパッチ。
- `SPEC.md` §5.1/§6.1.2/§6.2/§7/§9/§12 を新仕様へ更新。`README.md`/`CHANGELOG.md`/`docs/history`。

## 方針

- 課題1のズーム: Web Mercator 解像度 `mpp=156543.03392*cos(lat)/2^z` を用い、円の直径が
  ビューポートの約80%に収まる最大ズームを採用（floor(log2(...))、0〜19 にクランプ）。
- 課題2: `filter_company` の結果を `to_csv(index=False).encode("cp932", errors="replace")`。
- 課題4: `pages/` を使わないことが本質的な解（ディレクトリ名を `views/` に変更）。

## 役割分担

- メイン直接: `lib/data.py`（zoom_for_radius / filter_company）と両 caller のズーム呼出。
- 委譲（implementer-sonnet）: pages→views 改名・bulk 分割・app.py・zip_builder 整理。
- SPEC/README/CHANGELOG/docs はメインが更新。各段階で差分監査。

## 検証方法

1. `zoom_for_radius` 検算（円直径がビューポート約0.8倍に収まる）。
2. `render_static_map` を radius 2/5km で生成し、円が画像内に収まることを目視。
3. `filter_company` → cp932 CSV のエンコード/デコード確認。
4. `streamlit run app.py` でラジオ4項目・自動ページ一覧が出ないこと・各画面動作を確認。
5. Databricks 再デプロイ（`--full` 同期で旧 `pages/` 除去）。
