# 結果: 取得件数サマリを店舗数のみの表示に変更

- 日時: 2026-07-09 01:53
- テーマ: fetch-count-stores-only
- 計画: [20260709-0153-fetch-count-stores-only-plan.md](20260709-0153-fetch-count-stores-only-plan.md)

## 変更内容

- `views/main_page.py` — 取得件数サマリの文言を変更。
  - 非0件: `取得結果: {行数}件（店舗数: {店舗数}）` → `取得店舗数: {店舗数:,}件`。
    行数（店舗 × 推進園の組合せ数）の表示を廃止。
  - 0件: 冒頭を `取得結果: 0件 — …` → `取得店舗数: 0件 — …` に変更
    （企業名・半径・「取得処理は成功しています」の説明は従来どおり）。
  - `n_rows` 変数を削除し、空DFガードは `len(df)`、0件判定は `n_stores == 0` に統一。

## 検証結果

- `python -m py_compile views/main_page.py` — OK。
- ダミーDF（5行・2店舗）＋メッセージ生成ロジックの単体確認:
  - 非0件 → `success` / `取得店舗数: 2件`。
  - 0件 → `warning` / `取得店舗数: 0件 — 企業X / 半径2.0km に該当するデータが
    ありませんでした（取得処理は成功しています）`。
