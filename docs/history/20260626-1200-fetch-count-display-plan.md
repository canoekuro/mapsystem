# 計画: データ取得件数の表示

- 日時: 2026-06-26 12:00
- テーマ: fetch-count-display

## 背景・目的

「データ取得」ボタン押下時に Databricks からデータを取得しているが、取得件数を
一切表示していない。データが0件の場合、小売店名称 selectbox の選択肢が空になるだけで、
ユーザーは「クエリ成功・該当0件」なのか「取得処理失敗」なのかを区別できない。

データは「店舗 × 推進園」の組合せで1店舗に複数行存在する（SPEC §4.3）ため、取得した
**行数**と選択肢になる**店舗数**の両方を表示し、取得成功と0件を明確に区別できるようにする。

## 対象ファイル

- `views/main_page.py` — 取得件数サマリ表示ブロックの追加。

（`lib/data.py` は変更しない。件数は取得済みDFから算出する。）

## 変更内容

`render()` 内、取得済みDFを取り出した直後（`df = st.session_state["loaded_df"]` の後）に、
取得後は常時表示する取得件数サマリを追加する。

- `n_rows = len(df)` / `n_stores = df["店舗名称"].nunique()`。
- 0件: `st.warning("取得結果: 0件 — … 該当するデータがありませんでした（取得処理は成功しています）")`。
- 非0件: `st.success("取得結果: {n_rows:,}件（店舗数: {n_stores:,}）")`。
- 取得失敗（例外）は `load_filtered` 内で送出され Streamlit 標準エラーとなるため、
  本サマリが出る＝取得成功、という前提が成立する。
- 施設リスト側の0件 warning（店舗選択後・表示半径での絞込結果）は目的が異なるため残す。

## 検証方法

- `python -m py_compile views/main_page.py`。
- 0件・非0件ケースの表示メッセージ確認（ダミーDF＋streamlitスタブ）。
- 取得後に店舗選択・表示半径変更してもサマリが消えないこと。
