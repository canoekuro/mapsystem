# plan: 列名変更対応・出荷実績の表示（issue 202607231113）

## 背景 / 目的

Databricks 側テーブル `immune_promo_nursery` / `immune_promo_store` が更新され、
(1) `企業G名称` 追加・`店舗コード`→`小売店CD` 改名、(2) 3商材（プラズマ計 / おい免 /
ムテキッズ）の「当年実績（箱数）/ 前年実績（箱数）/ 前年比」列追加、(3) 出荷実績の対象期間を
記した `rdp_update.toml` の Volume 配置、が行われた。アプリをこれに追随させ、実績と期間を
マップ画面・店舗別 推進園数テーブルへ反映する。

## 変更対象と内容

1. `config/column_mapping.toml`
   - `"店舗コード" = "小売店CD"` に張替（アプリ内部名は据置。`_rename_to_app_columns` が全経路で戻す）。
   - `企業G名称` と 3商材×3指標の9列を恒等エントリで追加。`業態名称` は未使用のため残置。

2. `config/databricks_config.toml`
   - `[rdp]` 追加: `update_toml_path`（Volume）／`local_fallback = docs/issues/rdp_update.toml`。

3. `lib/data.py`
   - `SALES_PRODUCTS` / `SALES_METRICS` / `SALES_COLUMNS` / `sales_column()` を定義。
   - `load_stores` の SELECT に `企業G名称` と実績9列を追加。
   - `store_nursery_counts` の出力へ実績9列を追加（存在する分のみ、列順は 推進園数 の後）。
   - `load_shipment_period()` 新設。Volume の `rdp_update.toml` を WorkspaceClient で取得（失敗時
     ローカルfallback）、正規表現で `（開始 ～ 終了）` を解析し `開始～終了` を返す。`@st.cache_data(ttl=300)`。

4. `views/main_page.py`
   - `_sales_table_html(srow, period)` へ変更。実績（箱数）はDB値そのまま、前年比は ×100 して
     小数1桁＋`%`、欠損は `—`。右下に `出荷実績　期間：{period}`。
   - 呼び出しを `_sales_table_html(srow, load_shipment_period())` に更新、import 追加。

5. サンプル / ドキュメント
   - `scripts/gen_sample.py`：`企業G名称`・実績9列（店舗単位で一定）を追加し `data/master.csv` を再生成。
   - `SPEC.md` §4.2 カラム定義・§6.1.2 出荷実績テーブルを追記。
   - `docs/history/`（本plan/result）・`CHANGELOG.md`。

## 整形の確定事項

- 前年比＝比率（例 1.053）→ ×100・小数1桁・`%`（→ `105.3%`）。実績（箱数）＝DB値そのまま。

## 検証

- `python -c "import lib.data, views.main_page"` の import 成功。
- `load_shipment_period()` がローカルfallback から `2026年06月～2026年06月` を返す。
- `store_nursery_counts` が実績9列付き（欠損時も壊れない）で返る。
- `_sales_table_html` の値整形（そのまま / ×100%% / 欠損—）と期間表記。
