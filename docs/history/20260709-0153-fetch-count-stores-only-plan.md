# 計画: 取得件数サマリを店舗数のみの表示に変更

- 日時: 2026-07-09 01:53
- テーマ: fetch-count-stores-only

## 背景・目的

現在の取得件数サマリは `取得結果: {行数}件（店舗数: {店舗数}）` の形式で、行数
（店舗 × 推進園の組合せ数）が主として表示されている。ユーザーにとって意味があるのは
選択肢になる店舗数であり、行数は不要とのフィードバックを受け、表示を
`取得店舗数: {店舗数}件` のみに簡素化する。

## 対象ファイル

- `views/main_page.py` — 取得件数サマリの文言変更。
- `docs/history/20260709-0153-fetch-count-stores-only-{plan,result}.md` — 本記録。
- `CHANGELOG.md` — Unreleased 配下に1行追記。

## 変更内容

`render()` 内の取得件数サマリブロックを以下のとおり変更する。

- 非0件: `st.success("取得店舗数: {n_stores:,}件")`（行数の表示を削除）。
- 0件: `st.warning("取得店舗数: 0件 — {企業} / 半径{半径}km に該当するデータが
  ありませんでした（取得処理は成功しています）")`（冒頭の文言のみ変更）。
- `n_rows` 変数は表示に使わなくなるため削除し、`len(df)` を空DFガードに直接使用。
- 0件判定は `n_stores == 0` に統一（行数0 ⇔ 店舗数0 のため挙動は不変）。

## 検証方法

- `python -m py_compile views/main_page.py`。
- ダミーDF＋streamlitスタブで 0件・非0件それぞれのメッセージを確認。
