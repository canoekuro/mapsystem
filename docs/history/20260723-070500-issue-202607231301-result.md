# 結果: issue 202607231301（RDPアップロード・DL統合・本部担当用ページ・距離桁数）

## 変更内容

### 要件1: RDPアップロード追加 + 削除スコープ化
- `lib/volume.py`: `replace_in_volume(dir_path, filename, base, data)` に `base` 引数を追加。フォルダ内全削除をやめ、`{base}.` で始まるファイルのみ削除するよう限定。同一フォルダに複数ベース名（nursery/store/rdp）が同居しても互いを消さない。
- `views/upload_page.py`: `_TARGETS` に `("RDPデータ（xlsx/xls）", "rdp_dir", "rdp", "up_rdp")` を追加。`replace_in_volume` へ `base` を渡すよう更新。ヘッダ文言に RDP を追加。列は `_TARGETS` 数連動のため自動で3列化。
- `config/databricks_config.toml`: `[volume]` に `rdp_dir`（他2つと同一フォルダ）を追加。

### 要件2: 商談資料・店舗POP DL統合
- `lib/zip_builder.py`: `build_pptx_zip(files: dict[str, bytes]) -> bytes` を追加。
- `views/main_page.py`: 「商談用資料」「店舗POP」の2ボタンを「商談資料・店舗POPダウンロード」1ボタンへ統合。押下で両 pptx を `{store}_商談資料・店舗POP.zip`（mime `application/zip`）にまとめてDL。

### 要件3: ページ再編（店舗担当用／本部担当用）
- `app.py`: 「マップ」→「店舗担当用」に改称。`本部担当用`（`_hq_page`）を登録。`load_company_group_names` を startup で読む。
- `lib/data.py`: `load_company_group_names` / `load_stores_by_group` / `load_filtered_by_group` を追加。`store_nursery_counts` に `include_company: bool=False` を追加（True で先頭に企業名称列。企業名称も集計・結合キーに含め、企業跨ぎの店舗名衝突を回避）。
- `views/hq_page.py`（新規）: sidebar に企業G名称・取得半径・データ取得・DLボタン、メインに店舗別 推進園数の表（企業名称列を含む）。マップ・施設リスト・出荷実績・pptx は無し。出典フッタは `main_page._data_source_caption` を再利用。

### 要件4: 距離表示 2桁→1桁
- `views/main_page.py`（施設リストカード）・`lib/map_builder.py`（地図ツールチップ）・`lib/png_builder.py`（PNG出力）の `距離 .2f`→`.1f`。
