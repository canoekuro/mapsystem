# 計画: issue 202607231301（RDPアップロード・DL統合・本部担当用ページ・距離桁数）

## 目的・背景

`docs/issues/202607231301.md` の4要求に対応する。

1. データ更新に RDP ファイルが増えた。既存2ファイル（推進園/店舗）と同様のアップロードIFを用意し、他2ファイルと同じ Volume フォルダにベース名 `rdp` で格納する。
2. 商談用資料と店舗POP は両方DLするケースが大半のため、「商談資料・店舗POPダウンロード」1ボタンに統合し、押下で両方をDLできるようにする。
3. 「マップ」ページを「店舗担当用」に改称し、「本部担当用」ページを新設。本部用は企業G名称と半径を指定して取得すると店舗別 推進園数の表を表示・DLできる（マップ不要）。
4. 施設リスト等の距離を小数2桁（約0.56km）→小数1桁（約0.6km）へ変更。

## 方針（ユーザー確認済み）

- アップロード削除はベース名単位（`rdp.*` / `nursery.*` / `store.*` のみ削除）に変更し、3ファイルが同一フォルダで共存できるようにする。既存の潜在バグ（同時アップロードで片方消失）も解消。
- 距離1桁化は全箇所（施設リスト・地図ツールチップ・PNG/pptx出力）に統一。
- 本部担当用テーブルには企業名称列を追加（企業Gは複数企業を跨ぐため）。

## 対象ファイル

- `lib/volume.py`: `replace_in_volume` を base 名スコープ削除に変更（引数 `base` 追加）。
- `views/upload_page.py`: `_TARGETS` に RDP 追加、`replace_in_volume` に `base` 渡し、文言更新。
- `config/databricks_config.toml`: `[volume] rdp_dir` 追加。
- `lib/zip_builder.py`: `build_pptx_zip(files)` 追加。
- `views/main_page.py`: 2 DLボタンを ZIP 1 ボタンへ統合、距離 `.2f`→`.1f`。
- `lib/data.py`: `load_company_group_names` / `load_stores_by_group` / `load_filtered_by_group` 追加、`store_nursery_counts(include_company=)` 拡張。
- `views/hq_page.py`（新規）: 本部担当用ページ。
- `app.py`: ページ改称・`本部担当用` 登録。
- `lib/map_builder.py` / `lib/png_builder.py`: 距離 `.2f`→`.1f`。
- `SPEC.md` / `CHANGELOG.md` / `docs/history/`。

## 検証方法

- `python -m py_compile` で全 py の構文チェック。
- pandas による純ロジック検証（`store_nursery_counts(include_company=True)` の企業名称列・0件店舗保持・企業跨ぎの店舗名衝突分離、`build_pptx_zip` の同梱、距離 `.1f` フォーマット）。
